Read the [release channel contract](/docs/reference/release-channels) before an
upgrade. Stable describes clean-room-tested public npm artifacts; Next describes
the current repository surface.

Before changing production dependencies, reproduce the target version in a
clean project and run the paid seller/buyer transaction. Do not infer npm
availability from a package version in the repository.

For defects or ambiguous behavior, include the exact Lucid and x402 package
versions in a [GitHub issue](https://github.com/daydreamsai/lucid-agents/issues).
